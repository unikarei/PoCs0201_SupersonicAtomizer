# Droplet Transport Validation Cases

Required droplet transport validation cases:

- **Zero-Slip or Near-Zero-Slip Case**
- **Slip-Driven Acceleration Case**

Each case should include example YAML inputs and expected slip, velocity, and diameter trends.

- The Slip-Driven Acceleration Case should demonstrate **drag-driven acceleration** behavior where aerodynamic drag causes droplets to accelerate toward gas velocity.

- For these droplet-transport validation cases, explicitly note when breakup effects are not required (e.g., breakup effects not required) so gas-only droplet trends can be isolated.

- Validation outcomes for droplet-transport cases should be reported with a pass/warn/fail status for CI and reviewer checks (pass/warn/fail status).

- Each droplet-transport validation case should include a short qualitative trend description for reviewer guidance (qualitative trend).
